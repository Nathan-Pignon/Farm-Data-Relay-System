This is a driver for SSD1306 128x64, 128x32, 64x48 and 64x32 OLED displays running on the Arduino/\+ESP8266 \& ESP32 and mbed-\/os platforms. Can be used with either the I2C or SPI version of the display.

This library drives the OLED display included in the \href{https://thingpulse.com/product/esp8266-iot-electronics-starter-kit-weatherstation-planespotter-worldclock/}{\texttt{ Thing\+Pulse IoT starter kit}} aka classic kit aka weather station kit.

\href{https://thingpulse.com/product/esp8266-iot-electronics-starter-kit-weatherstation-planespotter-worldclock/}{\texttt{ }}

You can either download this library as a zip file and unpack it to your Arduino/libraries folder or find it in the Arduino library manager under "{}\+ESP8266 and ESP32 Oled Driver for SSD1306 display"{}. For mbed-\/os a copy of the files are available as an mbed-\/os library.

It is also available as a \href{https://platformio.org/lib/show/2978/ESP8266\%20and\%20ESP32\%20OLED\%20driver\%20for\%20SSD1306\%20displays/examples}{\texttt{ Platform\+IO library}}. Just execute the following command\+:

Fonts are defined in a proprietary but open format. You can create new font files by choosing from a given list of open sourced Fonts from this web app\+: \href{http://oleddisplay.squix.ch}{\texttt{ http\+://oleddisplay.\+squix.\+ch}} Choose the font family, style and size, check the preview image and if you like what you see click the "{}\+Create"{} button. This will create the font array in a text area form where you can copy and paste it into a new or existing header file.

The library supports different protocols to access the OLED display. Currently there is support for I2C using the built in Wire.\+h library, I2C by using the much faster \href{https://github.com/pasko-zh/brzo_i2c}{\texttt{ BRZO I2C library}} written in assembler and it also supports displays which come with the SPI interface.\hypertarget{md_src_2_e_s_p8266__and___e_s_p32___o_l_e_d__driver__for___s_s_d1306__displays_2_r_e_a_d_m_e_autotoc_md17}{}\doxysubsubsection{\texorpdfstring{I2C with Wire.\+h}{I2C with Wire.\+h}}\label{md_src_2_e_s_p8266__and___e_s_p32___o_l_e_d__driver__for___s_s_d1306__displays_2_r_e_a_d_m_e_autotoc_md17}

The Ui Library is used to provide a basic set of user interface elements called {\ttfamily Frames} and {\ttfamily Overlays}. A {\ttfamily Frame} is used to provide information to the user. The default behaviour is to display a {\ttfamily Frame} for a defined time and than move to the next {\ttfamily Frame}. The library also provides an {\ttfamily Indicator} element that will be updated accordingly. An {\ttfamily Overlay} on the other hand is a piece of information (e.\+g. a clock) that is always displayed at the same position.
